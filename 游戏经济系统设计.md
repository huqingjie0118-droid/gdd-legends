# 游戏经济系统设计文档（GDD v1）

> 适用项目：传奇 H5 ARPG（Canvas 2D 单文件 / 可转微信小游戏）
> 单一数据源：`BAL` 常量（`index.html`）；关联文档：《游戏机制与系统设计》《任务与奖励机制系统设计》《战斗系统深化设计》
> 本文金币数值**均为已含 `REWARD_SCALE:7` 的展示值**（与既有怪物掉落口径一致）；未实测项标 `[PLACEHOLDER]`。

## 0. 核心理念：经济不是"抽水机"，而是"水泵"

很多游戏经济设计把**消耗**当成"把金币倒进黑洞"，结果要么玩家舍不得花（囤积→隐性通缩），要么花完没回报（挫败→弃游）。

我们的定义：**好的消耗是"水泵"——花金币提升战力，战力抬高你的产出天花板，于是你能赚更多金币，于是又想花**。获取与消耗必须形成**自强化闭环**，而不是单向漏斗。

```
        ┌─────────── 消耗(水泵) ───────────┐
        │                                     ▼
   ┌────┴────┐                          战力↑ / 外观↑
   │  金币    │                              │
   │  获取    │◄──── 产出天花板↑ ◄──────────┘
   └────┬────┘        (能打更高难度的内容)
        │
   多种获取途径(按玩家类型分流)
```

**设计三铁律**
1. 每一笔大额消耗，都要**回流成更高的 earning capacity**（否则它是死 sink）。
2. 货币**分层**：软货币(金币)无限刷、绑金防 RMT 泄漏、代币周清零、硬货币(灵玉)由鲸鱼贡献——任何一层通胀都不致全盘崩溃。
3. ** sinks 必须随货币供给缩放**（物价随服均等级抬升），否则后期金币变废纸。

## 1. 货币体系（四层防通胀）

| 货币 | 性质 | 主要来源 | 主要 sink | 防滥用设计 |
|---|---|---|---|---|
| **金币** | 软货币，无限可刷 | 怪物 / 任务 / 贸易 / 竞技 | 强化 / 重铸 / 传送 / 复活 / 修理 / 外观(廉价) | 主循环货币 |
| **绑金** | 绑定软货币 | 怪物掉落(部分)、绑定性任务 | 仅限"非交易类" sink（强化/修理） | 不可交易 / 不可上拍卖行 → 切断 RMT 提现链 |
| **代币** | 限时活动币 | 每日 / 周常（见任务文档） | 周常商店(限时外观/材料) | **每周清零** → 天然无通胀、制造紧迫 |
| **灵玉** | 硬货币(稀缺) | 成就巅峰奖励 / 直接获取(鲸鱼) | 专属外观 / 称号 / 便利(仓库格) | 不可农场量产 → 保住付费价值 |

> **为什么必须分层**：若只有金币一种，农场号刷爆会拉崩所有物价；把"稀缺价值"锁在灵玉、"时间价值"锁在代币、"自由流通价值"留给金币，三层互不影响，单层波动不致系统性崩盘。

## 2. 金币获取途径（按玩家类型分流）

目标：**战斗型 / 任务型 / 贸易型 / 社交型 / 竞技型** 五类玩家各有主路，谁都能赚钱。

### 2.1 战斗型（最主流）
| 途径 | 公式（展示值） | 节奏 |
|---|---|---|
| 普通怪掉落 | `GOLD_BASE(=56@L1) × levelFactor(L) × mapFactor` | 主被动收入 |
| 精英怪 | × `ELITE_GOLD(=2.0)` | 地图内尖峰 |
| Boss | × `8~15 [PLACEHOLDER]` | 周常/世界事件 |
| 环境互动宝箱 | `30~300` 随地图 | 探索红利 |

- `levelFactor(L) = clamp(1 + BAL.MON_LEVEL_SCALE×(L-1), 0.75, 2.2)`（复用 `BAL.MON_LEVEL_SCALE=0.045`）
- `mapFactor = 1 + BAL.MAP_DIFF_STEP × mapIndex`（`MAP_DIFF_STEP=0.05`）
- **碾压衰减**（与任务文档一致）：当 `playerLevel − 内容等级 ≥ 5`，金币 ×0.6，防"回新手村刷毕业"。

### 2.2 任务型（稳定被动）
- 复用《任务与奖励机制》金币公式：`questGold = BASE_GOLD[d] × 1.18^(L-1)`（`BASE_GOLD[D1..D5]=40/80/160/320/640`）。
- 每日 D1–D2 是"保底 passive income"，让休闲玩家**离线一天也不掉队太多**，但上限低，逼不出毕业。

### 2.3 贸易型（非战斗玩家的活路）
- **拍卖行**：玩家间低买高卖（材料 / 装备 / 词缀），赚取差价；系统收 `5% [PLACEHOLDER]` 手续费（既是 sink 也抑制通胀）。
- **生活技能**：采集(草药/矿石) + 制造(药水/强化石/符文)，产出的**消耗品**是永远的需求（战斗型也要买药），形成"生产→消费"跨玩家循环。
- **倒爷红利**：版本更新 / 新 Boss 前囤某材料，更新后高价出 → 市场嗅觉变现。

### 2.4 社交型
- **组队分红**：周常 `team` 任务每人奖励 ×1.15（任务文档），组队本身提高单位时间产出。
- **带人/助战**：高玩带低玩通关高难内容，低玩付费(金币)雇援，社交关系变现（限游戏内金币，禁 RMT）。
- **公会捐赠 / 建设**：捐金币换公会 Buff（全团经验 +5% [PLACEHOLDER]），集体目标。

### 2.5 竞技型
- **排行榜**周结算（等级/战力/成就/速通/组队贡献榜）前 N% 给金币 + 限时外观。
- **速通 / 无伤挑战**成就给一次性大额金币 + 称号。

> **设计落点**：五类玩家赚钱主路互不挤压——战斗靠时间、贸易靠脑子、社交靠关系、竞技靠技术、任务靠习惯。没有"唯一最优赚钱法"，符合反单一最优原则。

## 3. 金币消耗渠道（让玩家"花得值"）

每个 sink 都标注**心理动因**（行为经济学， Ethical 应用）：玩家觉得值，才愿意花，循环才转得动。

### 3.1 战力泵（消费即提产——核心 sink）
| 消耗 | 公式（展示值） | 为何花得值 |
|---|---|---|
| 装备强化 +N | `enhCost(N)=BASE_ENH × 1.25^N × (1+0.07×equipLevel)`，`BASE_ENH=200 [PLACEHOLDER]` | 每次 +N 即时跳字 + 辉光增强（**胜任感**）；+15 是终极目标 |
| 词缀重铸 | 每次 `refCost=BASE_REF(=500) × 重铸次数档` | **可变比率奖励**式的赌博快感（roll 出极品词缀），配 pity 防上头 |
| 镶嵌 / 附魔 | 孔位费 + 宝石价 | 自定义 Build（**自主性**） |
| 洗练属性 | 按重洗次数递增 | 实验不同流派 |

- `enhCost` 指数增长 → **后期强化是最大金币黑洞**，正好吞掉后期暴涨的收入，是天然 anti-inflation。
- 例（已用 `BASE_ENH=200, 1.25^N` 核验）：+1≈250、+5≈610、+10≈1863、+15≈5684 金；累计 +15 ≈ **2.7 万金**，而后期 D5 周常单次可产数千金 → 收支咬合。

### 3.2 便利与保险（损失厌恶）
| 消耗 | 动因 |
|---|---|
| 传送费 | 省时间 = 被估值（**便利性**） |
| 复活费 | 避免掉落惩罚（**损失厌恶**：花小钱避大损） |
| 修理费 | 装备耐久损耗（持续 sink） |
| **保护券**（强化不碎） | 损失厌恶的"购买安心"——小额预付规避大额风险 |

### 3.3 身份与表达（自主性 + 地位）
| 消耗 | 动因 |
|---|---|
| 外观 / 染色 / 武器特效 | 身份表达（**关系感 / 地位**，Cialdini 稀缺性） |
| 坐骑 / 宠物 | 兼具战力与排面 |
| 专属称号(灵玉为主，金币为辅) | 荣誉 sink |

### 3.4 社交与公会
- 公会捐赠 / 建设（换集体 Buff）、好友互助礼金 —— **关系感**驱动。

### 3.5 重置与探索
- **洗点 / 洗技能**：让玩家自由实验流派（**自主性**），小额多次 sink。
- **转生消耗**：花钱 + 花时间换永久增益 + 重置 → 长线循环发动机。

> **伦理约束（负责任设计）**：词缀重铸属"可变比率"机制，易诱发沉迷——必须配 **pity 保底**（连续 N 次未出极品则加权），并对单日重铸次数设**软上限冷却**；灵玉(付费层)绝不给"独占战力"只给"便利/外观"，保证免费玩家靠时间可达同等强度（不 pay-to-win 卡核心进度）。

## 4. 正向循环引擎（本文核心）

把 §2 来源与 §3 sink 拼成闭环，**关键是 sink 必须抬升 earning ceiling**：

```
玩家花金币强化/重铸(§3.1)
   → 战力↑、词缀极品
   → 能稳定刷 每日 D4/D5、周常 Boss、精英(§2.1 高倍率)
   → 单位时间金币产出 ×2~×4(精英×2、D5 任务×4)
   → 金币更多 → 又想强化下一档 / 洗下一词缀
   → 持续参与 ✓
```

- **为什么是"正向"而非"无底洞"**：在死 sink 模型里，花掉就没了，玩家越花越穷→焦虑→停手；在水泵模型里，花掉换来"赚得更多的能力"，玩家越花越能玩→爽→接着玩。
- **双速设计**：
  - 低等级段：**来源 > 消耗**（新手慷慨，先让玩家富起来、建立"金币有用"认知）。
  - 高等级段：**消耗略 > 来源**（温和压力，制造"还差一点就能 +15"的继续动力，但不至于卡死）。
- **版本抬升天花板**：每次大版本提高战力上限 + 开新地图(更高 `mapFactor`) → 旧 sink 填不满新天花板 → 新一轮"赚→花→更强"启动，对抗内容消耗殆尽后的倦怠。

## 5. 平衡与通胀 / 紧缩防控

### 5.1 通胀监测指标（硬告警）
- **核心指标**：`人均日金币净增 = (日来源合计 − 日消耗合计) / 活跃玩家数`。
- 连续 **7 日** 净增 > `NET_INFLATION_THRESHOLD [PLACEHOLDER]` → 自动触发**平衡 pass**（升物价 / 加 sink / 削某来源）。
- 反向：**通缩告警**：若中位数玩家在当前等级**买不起对应档强化**（金币枯竭）→ 提怪物/任务金币。

### 5.2 动态 sink 缩放（物价随钱潮涨）
- `priceMult = clamp(1 + 0.03 × (serverAvgLevel − 10), 1.0, 3.0)`。
- 全服越富，强化石 / 重铸单价越贵 → ** sinks 随货币供给同步长大**，从根上防废纸化。这是分层之外的第二道通胀闸门。

### 5.3 供给 / 需求均衡曲线
- 绘制**单 session**（0–60min）与**单周**的 supply(§2) vs demand(§3) 曲线。
- 设计目标：session 内早期供>需（爽），session 末需>供（有目标）；周维度维持微缺口（持续动力）。
- **Monte Carlo** 模拟 1 万玩家随机行为，检测极端路径（纯农场号、纯倒爷、纯鲸鱼）下的金币存量分布，找崩盘边界 `[PLACEHOLDER]`。

### 5.4 其他阀门（汇总）
- 碾压衰减（§2.1，等级差≥5 金币×0.6）
- 绑金分流（切断 RMT 提现）
- 代币周清零（时间价值不囤积）
- 拍卖行手续费（5% 兼 sink）
- 神话掉率恒 `2/1000` 权重（装备不泛滥→强化需求永续）

## 6. 长期动力（留存）

| 机制 | 作用 |
|---|---|
| 版本抬升战力天花板 + 新地图 | 重启"赚→花"循环，抗内容倦怠 |
| 赛季 / 限时排行 | 每赛季重置部分(非金币)进度，制造新目标 + 稀缺外观 |
| **转生**：花金币+时间换永久增益+重置 | 长线"再练一轮"发动机 |
| **图鉴 / 收藏**：集齐给奖励 | 禀赋效应 + 沉没成本，留住核心玩家 |
| 公会 / 好友关系链 | 关系感绑定，单人难弃游 |

## 8. 落地方案（P0–P3，对接现有系统）

- **P0（循环跑通）**：怪物金币掉落（挂 `BAL.MON_LEVEL_SCALE/MAP_DIFF_STEP`）+ 强化 +N 系统（指数 `enhCost`）+ 修理费 + 传送费。立刻让"赚→花→更强"转起来。
- **P1（分层与防漏）**：绑金分流、拍卖行 + 5% 手续费、代币周清零（复用任务文档）。
- **P2（水泵加深）**：词缀重铸(可变比率 + pity)、洗点/洗技能、外观/坐骑/宠物商店。
- **P3（闭环调控）**：`priceMult` 动态物价、净增监测告警、Monte Carlo 边界检测、转生消耗、图鉴收藏。

## 10. 一句话总结

经济系统的成败不在"印多少金币"，而在**花掉金币是否让玩家变得更能赚**。我们把消耗设计成水泵（强化/重铸抬升产出天花板）、把货币分成四层（金币/绑金/代币/灵玉）各自封住通胀漏洞、用动态物价和净增监测做第二道闸门——于是"赚金币→变强→赚更多金币"成为自驱动的长期动机，而非策划手动喂糖的短暂快感。

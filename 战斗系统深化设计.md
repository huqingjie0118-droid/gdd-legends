# 传奇H5 ARPG —— 战斗系统深化设计（深度 & 趣味性）

> 版本：v1.0 · 作者：GameDesigner · 状态：设计稿（数值含 `[PLACEHOLDER]`，需实测）
> 目标：解决「不停砍怪很无聊」——为战斗注入**多层决策**，让每次交战都有"我在做选择"的快感。
> 配套：游戏机制与系统设计.md / 技能树系统设计文档.md / 数值平衡调整说明.md（共用 `BAL` 数据源）。

---

## 0. 问题诊断：为什么现在无聊

现有循环只有一层：**锁定 → 普攻 → 跳字**。玩家在战斗中不需要思考，只需重复动作。无聊的根源是**决策缺失**：

| 缺失维度 | 表现 | 本设计补什么 |
|---|---|---|
| 节奏决策 | 没有"何时放何技"的取舍 | 连招 / 技能组合 / 元素反应 |
| 目标决策 | 怪物都是铁板一块 | 弱点 / 破绽 / 打断 / 部位破坏 |
| 空间决策 | 站桩即可 | 环境互动 / 地形机制 |
| 构建决策 | 装备随便穿 | 套装协同 / 词缀联动 |
| 路线决策 | 加点无方向 | 流派 / 觉醒分支 |

**设计信条**：深度 ≠ 复杂数值，而是"更多有意义的抉择 + 抉择有清晰反馈"。

## 1. 连招系统（Combo / 节奏决策）

### 1.1 连击计数与衰减
- 每次命中 +1 连击；`2.5s` 内未命中则连击归零（窗口随攻速/精通延长至 `[PLACEHOLDER]`）。
- 三职业"计数方式"不同但统一显示：战士(近战命中)、法师(技能命中)、道士(本体+召唤物命中)。

### 1.2 连击阶梯（阈值触发增益）

| 连击数 | 效果 | 设计意图 |
|---|---|---|
| ≥5 | 伤害 +8% | 起手奖励，鼓励不中断 |
| ≥10 | 伤害 +15% + 回少量资源 | 维持循环的甜头 |
| ≥20 | 伤害 +25% + 暴击率 +10% | 高风险高回报区 |
| ≥30 | 解锁「终结技」提示(按键) | 高潮点，消耗连击换大招 |

- **终结技(Finisher)**：高连击时屏幕提示 `终结!` 按键释放，消耗全部连击造成 `连击数 × k [PLACEHOLDER]` 的爆发，并重置连击。把"刷连击"转化为"何时收网"的决策。

### 1.3 防刷连击的阀门
- 连击增益对 Boss 衰减（Boss 韧性高，连击增益 ×0.4），避免连击流通吃所有内容。
- 连击窗口与攻速软上限(2.5/s)耦合，超阈攻速不增加有效连击。

## 2. 技能组合 & 元素反应（Skill Combo / 决策核心）

### 2.1 技能标签（Tags）
每个技能标 1~2 个标签：`火/冰/灵/地(物理)`、`控制/破甲/dot/击退/治疗`。组合玩法全靠标签联动。

### 2.2 元素反应（Elemental Reaction）
目标身上存在某元素附着时，施加**不同**元素触发反应（窗口 `3s`）：

| 反应 | 组合 | 效果 | 最佳场景 |
|---|---|---|---|
| 融化 Melt | 火 + 冰 | 一次性大额爆发伤害 | 单体 Boss |
| 过载 Overload | 火 + 灵 | 范围爆炸 + 击退 | 群怪 |
| 超导 Superconduct | 冰 + 灵 | 目标降防 40% 持续 6s | 高防单位 |
| 裂甲 Shatter-Armor | 地 + 火 | 破甲 + 灼烧 DoT | 重甲 |
| 冻土 Frostfield | 冰 + 地 | 减速 50% + 持续伤害 | 控场 |
| 蓄能 Charge | 灵 + 灵 | 充能速度翻倍 3s | 续航 |

> 设计意图：反应强制"技能顺序规划"——先挂冰再补火打融化，比无脑火火火高 30%~60% 输出。**但**反应收益随敌人抗性浮动（见 §3），故没有"永远最优"的起手。

### 2.3 状态连锁（Status Chains）
非元素的状态也能连锁：
- **破甲 → 处决**：破甲状态下重击触发处决，伤害 ×2.5 且回资源。
- **控制 → 易伤**：冰冻/眩晕中目标受到伤害 +50%。
- **dot 叠加 → 引爆**：灼烧/中毒叠满 N 层可被"引爆"技一次性结算，奖励叠层玩法。

### 2.4 资源类型分化（沿用职业定位）
- 战士「怒气」：受击/命中积累，反应/处决消耗 → 近身越打越猛。
- 法师「元素充能」：施法积累，高费反应技消耗 → 站桩爆发。
- 道士「道法」：召唤/治疗积累，控场技消耗 → 持久战。

## 3. 怪物弱点 & 破绽机制（目标决策）

### 3.1 弱点 / 抗性标签
每只怪标 `伤害类型抗性`（物理/火/冰/灵，±% 减免）与 `状态抗性`（控制/dot/破甲 免疫或减半）。命中弱点时怪物头顶**亮起对应图标 + 跳字变金色**，给即时正反馈。

> 反单一最优：地图按"火抗图/冰抗图/物免图"混编，迫使玩家按图换技能或走混合 BD，杜绝一套技能通杀。

### 3.2 韧性 / 破防（Stagger & Break）
- 每个敌人有**韧性条**（Boss 更厚）。破甲/重击/控制类技能填充更快。
- 韧性满 → **破防状态**：眩晕 `1.5~3s`，期间受到伤害 ×2，战士可按键**处决**（秒杀小怪 / 对 Boss 造成最大生命 % 伤害）。
- 破防是"激进集火 vs 苟活"的抉择点：敢压韧性就换爆发窗口。

### 3.3 打断（Interrupt）
- 敌人施放强力技时有**读条/红光预警**；在读条期间用控制技（眩晕/冰冻）打断 → 取消其技能 + 攻击者获得资源奖励。
- 把"看到预警→反应打断"做成高阶玩家的操作红利。

### 3.4 部位破坏（Boss Part Break）
- Boss 有关键部位（巨龙之翼 / 魔君核心 / 巨魔臂锤），各有独立部位 HP。
- 摧毁部位 → 禁用该部位对应攻击 + 必掉额外材料/词缀。
- 制造"优先打哪"的战术分工（多人/多宠时尤其明显）。

## 4. 环境互动元素（空间决策）

每张地图有主题化可互动物，且**与元素反应联动**：

| 元素 | 互动物 | 效果 | 反应联动 |
|---|---|---|---|
| 通用 | 炸药桶 | 受击/火 → 爆炸 AOE 火伤 | 火点燃 → 连锁 |
| 火山图 | 熔岩池 | 站入持续火伤，但获「火伤+」buff | 火技能增幅 |
| 雪原图 | 冰柱 | 击碎 → 冰爆范围减速 | 冰反应扩大 |
| 洞窟图 | 毒沼/藤蔓 | 点燃→蔓延燃烧区 | 火→dot 场 |
| 通用 | 油渍/草丛 | 点火→燃烧蔓延 | 火→区域控制 |
| 通用 | 符文阵 | 踩踏触发增益/伤害机关 | — |
| 深渊图 | 裂隙能量 | 引怪入裂隙→即死小怪 | — |

> 设计意图：环境把"站桩输出"变成"走位利用"——引怪到油面点火、把 Boss 引到冰柱旁击碎控场。空间成为可博弈的第三维。

## 5. 装备搭配策略（构建决策）

### 5.1 套装协同（Set Synergy）
定义若干套装，按 `2/4/6 件` 触发递增效果，与"散件高词缀"形成取舍：

| 套装 | 2 件 | 4 件 | 6 件 |
|---|---|---|---|
| 烈焰 | +火伤 15% | 灼烧扩散至周围 | 引爆强化 + 反应伤害 +30% |
| 寒霜 | +冰伤 15% | 冰冻时间 +50% | 冻土范围扩大 + 减速加深 |
| 磐石 | +HP 12% | 受击反伤 | 破防时回血 + 减伤 |
| 迅影 | +攻速 10% | 连击窗口 +1s | 终结技伤害 +40% |

> 取舍：套装给"协同爆发"，散件给"极限词缀"。词缀预算(见系统文档)限制散件无法全满，故"套装满配 vs 散件极限"是长期构建争论点。

### 5.2 词缀联动（Affix Synergy）
特定词缀组合产生质变（非简单相加）：
- 暴击率 + 暴击伤害 → 暴击流（阈值后收益陡增，但有软上限）。
- 攻速 + dot → 叠层流（更多 hit = 更快叠满引爆）。
- 破甲 + 处决 → 处决流（对高韧 Boss 特化）。
- 低血触发护盾 + 吸血 → 狂战流（越残越猛）。

### 5.3 特效触发（On-Hit Procs）
装备带随机触发特效（击杀火环 / 低血护盾 / 受击反震），引入可控的"混沌深度"，让配装涌现意外 combo。

## 6. 角色成长路线选择（路线决策）

在既有「三分支×六层技能树」之上，定义**流派(Archetype)** 与 **觉醒(Awakening)**，让加点有方向感：

### 6.1 职业流派（示例）
| 职业 | 流派A | 流派B | 流派C |
|---|---|---|---|
| 战士 | 狂战(爆发玻璃) | 盾战(坦克控场) | 反击战(格挡反伤) |
| 法师 | 元素师(反应流) | 炮台(站桩AOE) | 时空(控制减速) |
| 道士 | 召唤师(多宠) | 毒道(DoT) | 辅道(治疗增益) |

- 流派 = 技能树分支的"推荐加点路线" + 适配核心 + 适配套装，给新手清晰目标，给老手"逆流派"实验空间。

### 6.2 觉醒（Awakening / 转职）
- 角色达 `Lv.30 [PLACEHOLDER]` 可觉醒，解锁**招牌技能**（如狂战「血怒」、元素师「元素共鸣」），强化该流派身份。
- 觉醒后开放"进阶词缀槽"，进一步分化 Build。

### 6.3 洗点与试错
- 洗点费用递增（沿用系统文档 `500×N^1.5`），首免 → 鼓励早期试错、后期定型的自然曲线。

## 8. 反单一最优 & 可读性（落到战斗）

- **弱点+抗性地图混编** → 无一套技能通杀。
- **反应收益随抗性浮动** → 起手顺序因地制宜。
- **破防/打断/部位破坏** 奖励操作而非堆数值 → 技术流有天花板。
- **环境按图主题** → 打法随场景变。
- **套装 vs 散件取舍** → 长线构建争论。
- **可读性**：连击数 HUD、反应跳字（融化!/过载!）、破防红光、弱点图标亮起、打断成功音效——每个决策都有**即时因果反馈**，玩家能"读懂"战场并据此调整。

## 9. 落地优先级与验证

**分阶段落地（避免一次堆爆复杂度）**：
1. P0（必先有趣）：连招系统 + 怪物弱点图标 + 破防处决 → 立刻让"砍怪"变"找破绽"。
2. P1：元素反应 + 技能组合连锁 → 决策层数质变。
3. P2：环境互动 + 套装/词缀联动 → 空间与构建深度。
4. P3：流派/觉醒 + 部位破坏 + 打断 → 长线路线与高阶操作。

**验证标准（每次测试前定义）**：
- 新手期能无设计师协助打出至少 1 次"破防→处决"和 1 次"元素反应"。
- 全图混编下，单一 BD（如纯火反应流）胜率 < 85%（证明弱点机制生效）。
- 玩家主观问卷："战斗中需要做选择" 同意率 > 70%（替代"只是砍"）。
- 早期修手感（hit-stop/跳字/反馈），后修数值。

## 0. 设计支柱（非妥协体验）

1. **BD 多样性 > 数值膨胀**：三职业 × 三分支技能树 × 装备词缀 × 四象核心，组合出数十种可行 Build；强弱由「克制关系 + 场景适配」区分，而非单纯堆数值。
2. **可读战斗（Readable Combat）**：每次命中都有清晰因果反馈（hit-stop 顿帧、跳字、状态条变更），玩家能"读懂"战场并据此决策。
3. ** earned Progression（赢得的成长）**：角色变强主要来自玩家决策（加点 / 配装 / 选核 / 强化），而非挂机时长；死亡有真实代价但可恢复，不惩罚探索欲。
4. **好奇循环（Curiosity Loops）**：地图、彩蛋 NPC、跨幕意象链提供"再看一眼"的钩子，支撑长线留存。

## 1. 核心玩法循环（Core Loop）

```markdown
# Core Loop: 传奇H5 ARPG

## Moment-to-Moment (0–30 秒)
- Action: 移动 → 锁定目标 → 普攻/放技能 → 拾取掉落
- Feedback: 命中 hit-stop(60~120ms) + 伤害跳字 + 音效；击杀有粒子/掉落实体
- Reward: 经验、金币、可能掉装备；即时"我又变强了"的正反馈

## Session Loop (5–30 分钟)
- Goal: 进入一张地图，完成主线/支线目标（reach/kill/collect/talk/boss）
- Tension: 资源压力（药水/蓝量有限）+ 越级风险 + 精英/Boss 尖峰
- Resolution: 清图/击杀 Boss → 获得核心/装备/材料 → 回城强化或传送下一张图
- Hook: 章节 flavor 短句 + 未探索区域/彩蛋 NPC 的"再看一眼"暗示

## Long-Term Loop (小时~数周)
- Progression: 升级解锁技能点 → 点满分支 → 集齐四象核心 → 推进六幕主线 → 转生(Meta)
- Retention: 每日/每周目标、赛季排行、深渊无限层、图鉴收集、称号成就
```

**Fun Hypothesis（必须先有趣的核心）**：近身砍怪、看到伤害跳字与顿帧、捡到发光的装备——这一秒一秒的循环本身要有爽感，次级系统（剧情/经济/Meta）只是放大它，而非依赖它。

## 2. 玩家角色成长路径（Progression Path）

### 2.1 角色等级与属性

- **等级上限**：`L99`（软上限思维：主线在 ~L22 收尾，L50/L70/L90 为转生门槛）。
- **每级发放**：属性点 `5` + 技能点 `1`；里程碑 L10/20/30/40/50 额外 `3` 技能点。
- **五项属性**（按职业权重分配）：

| 属性 | 代号 | 主受益职业 | 作用（每点） |
|---|---|---|---|
| 力量 | STR | 战士 | 物理攻击 +2，负重 +1 |
| 智力 | INT | 法师 | 魔法攻击 +2，魔法值 +8 |
| 敏捷 | AGI | 道士/通用 | 暴击 +0.3%，攻速 +0.5%，移速 +0.3% |
| 体力 | VIT | 战士/通用 | 生命 +20，防御 +1 |
| 精神 | SPI | 道士/法师 | 魔法 +10，生命回复 +0.5/s，治疗强度 +1% |

- **属性成长软上限**：单属性超过 `等级×2` 后收益递减（边际 ×0.5），防止全点一维的极端 BD。
- **经验曲线**（沿用数值平衡说明）：基数 `150`，每级 `×1.32`；L10≈5.2k，L22≈159k，L50≈[PLACEHOLDER]。

### 2.2 技能树（复用既有三分支×六层结构）

- 战斗(Combat)/生存(Survival)/精通(Mastery) 三主分支，每分支 T1–T6，每职业约 18 节点。
- **解锁依赖**：同分支 `T(n)` 需 `T(n-1)≥1`；终极 `T6` 需本分支 `T5` 满级 + 角色等级门槛（战斗 Lv.20 / 生存 Lv.25 / 精通 Lv.30）。
- **协同设计**：跨分支节点刻意制造联动（如 战斗·烈火强化 × 精通·冷却掌控 = 高频烈火），鼓励混搭而非强制专精。
- **洗点**：花费递增金币（首次免费，之后 `500 × 次数^1.5`），避免"一次点错终身废"的焦虑。

### 2.3 装备系统（词缀 + 强化 + 品质）

- **6 槽位**：武器 / 护甲 / 头盔 / 靴子 / 戒指 / 项链。
- **品质梯度**（倍率，影响基础值与词缀预算）：

| 品质 | 倍率 | 颜色 | 主要来源 |
|---|---|---|---|
| 普通 | 1.0 | 白 | 杂兵 |
| 优秀 | 1.3 | 绿 | 精英/任务 |
| 精良 | 1.7 | 蓝 | Boss/副本 |
| 史诗 | 2.2 | 紫 | 高级 Boss/深渊 |
| 传说 | 3.0 | 橙 | 世界事件/成就 |
| 神话 | 4.0 | 红 | 赛季榜/全服通告 |

- **词缀预算（Affix Budget）**：每件装备有预算点 `B = 品质倍率 × (1 + (等级-1)×7%) × 槽位系数`。词缀按其数值消耗预算，玩家在"堆攻击 / 堆生存 / 堆功能"间做取舍——**无法一件装备全满**，这是反单一最优的关键阀门。
- **强化（+0~+15）**：消耗强化石，每级 `+atk%/+def%/+hp%` 递增；费用 `base × 1.25^tier`；成功率随等级递减（可用「保护石」锁定不降级）。强化是后期战力上限的主要来源，也是金币/材料的主要消耗口（sink）。

### 2.4 四象核心（Build 定义器）

主线集齐 地/火/冰/灵 四核，可镶嵌于角色「核心槽」，赋予 Build 身份：

| 核心 | 效果 | 协同职业 | 错配惩罚 |
|---|---|---|---|
| 地(Earth) | +HP%，+DEF%，反伤 | 战士 | 错配职业效果 ×0.6 |
| 火(Fire) | +魔法攻击%，灼烧 DoT | 法师 | 同上 |
| 冰(Ice) | +控制/减速，+再生 | 道士 | 同上 |
| 灵(Spirit) | 全属性小幅+，技能急速 | 通用/混合 | 无惩罚（但单项不如专精核） |

> 设计意图：核心让"战士火核"之类的 off-meta 玩法**可行但不最优**，保留实验乐趣，又不会颠覆职业定位。

### 2.5 转生（Meta Progression）

- 在 L50 / L70 / L90 可转生：等级清零、保留装备与技能树，获得 **转生点** 投入 Meta 天赋树（全局增益：经验%+ / 掉率%+ / 起始资源+）。
- 转生制造"长线目标"与"重复可玩"，同时把数值天花板重新打开，避免满级后无事可做。

## 3. 资源管理系统（Economy）

```markdown
# Economy Balance（关键变量，单一数据源 = BAL 常量）
Variable                | Base      | Min   | Max   | Tuning Notes
------------------------|-----------|-------|-------|----------------------------------
玩家基础 HP(L1)         | 200       | 150   | 280   | 按职业微调 [PLACEHOLDER]
玩家攻击 ATK(L1)        | 20        | 15    | 28    | 战士高/法师低(靠技能)
金币掉落/击杀           | 5~15      | 1     | 50    | MON_SCALE 同步缩放
强化石掉率/地图         | 0.12      | 0.05  | 0.3   | 后期图更高

四象核心掉率            | 剧情固定  | -     | -     | 非随机，主线保送

洗点费用(第N次)         | 500×N^1.5 | 0(首免)|[PLACEHOLDER]| 防刷点
```

**Sources（来源）与 Sinks（消耗）平衡原则**：
- 金币来源：击杀/任务/卖装；消耗：修理、商店药、洗点、部分强化费。
- 强化石来源：地图掉落/活动；消耗：装备强化（主要 sink，制造长线追求）。
- 材料来源：分解装备/采集；消耗：词缀重铸、装备打造。
- **通胀监测指标**：`人均日金币增量`。超过阈值 `K_gold [PLACEHOLDER]` 触发一波平衡（下调掉率/加 sink）。
- **三档玩家（鲸/豚/鲫）**：鲸需要炫耀型 sink（神话外观/赛季榜）；豚需要性价比 sink（强化/词缀）；鲫需要可肝的 aspirational 目标（传说装备可刷得）。

## 4. 战斗与交互机制（Combat）

### 4.1 伤害公式（单一公式，覆盖普攻与技能）

```
基础伤害 = (ATK_or_MATK × 技能系数 × (1 + Σ增伤%) ) − DEF × 减防因子
最终伤害 = 基础伤害 × 随机浮动(0.9~1.1) × 暴击倍率(若暴击)
暴击倍率 = 暴击伤害% (基础 150%，上限 300%)
DEF 减伤 = DEF / (DEF + K)，K = 200 + 玩家等级×10   // 防御真正生效，非线性
```

- **攻速**：基础 1.0 次/秒，受 AGI 与精通分支影响，软上限 2.5 次/秒。
- **命中/闪避**：命中率 = clamp(攻击方命中 − 防御方闪避 + 基础, 0.6, 1.0)；高闪避 Build 存在但被"必中"词缀/技能克制。

### 4.2 三职业战斗定位（石头剪刀布式互克）

| 职业 | 定位 | 优势场景 | 被克 |
|---|---|---|---|
| 战士 | 近战坦克/单体爆发 | 站桩 Boss、物理群 | 法师 AOE 消耗 |
| 法师 | 远程 AOE/爆发 | 群怪、固定阵型 | 道士召唤物消耗 + 近身 |
| 道士 | 召唤/DoT/治疗 | 持久战、多线 | 战士突进秒杀宠 |

> 内容设计上**混编敌种**（物免/魔免/ swarm / Boss），使没有任何单一 BD 通吃所有内容 → 玩家需按图换装或走混合 BD。

### 4.3 反馈与手感（可读性）

- **hit-stop 顿帧**：命中 60~120ms（Boss 受击更长），让每一次伤害"落地有声"。
- **跳字**：白字普攻 / 黄字暴击 / 红字致命 / 绿字治疗，颜色即信息。
- **状态条**：Boss 多阶段血条带阶段标记（沿用既有实现）；玩家Buff/Debuff 图标化。
- **资源机制分化**：战士用「怒气」（受击/命中积累，放大爆发）；法师/道士用「魔法值」（技能消耗，靠 SPI 与回蓝词缀）；道士额外有「召唤物」承伤。

### 4.4 交互机制

- **NPC 对话**：`F` 键触发，按当前章节 `beats` 切换台词（已实现），彩蛋 NPC `when:'default'` 常驻。
- **环境交互**：传送阵（等级门槛 `minLevel`）、可破坏物（掉材料）、采集点。
- **死亡**：原地倒下→"你已陨落"结算→消耗部分金币/耐久原地复活（保装备），鼓励"再来一次"而非退游。

## 7. 系统间平衡与"反单一最优"（Anti-Single-Optimal）

目标：**不存在一个 Build 在所有场景都最优**。手段：

1. **职业互克**：物免怪克法师、魔免怪克战士、swarm 克单体、Boss 克脆皮 → 内容混编逼迫适应。
2. **核心错配惩罚 ×0.6**：off-meta 可行但不最优，保留实验又不颠覆定位。
3. **属性软上限**：堆一维收益递减，鼓励均衡或取舍。
4. **词缀预算**：单件无法全满，逼迫"要输出还是要活"。
5. **Build 成本曲线**（设计上人为制造差异）：
   - 玻璃大炮：前期便宜（堆攻即可），后期昂贵（需词缀/核心补生存）→  Early Game 强，Late Game 吃资源。
   - 坦克：前期贵（强化/血装烧钱），后期稳 → Late Game 强。
   - 混合：中庸，全程可用但上限低于 specialization → 新手友好、天花板低。
6. **PvE 多样性门槛**：每张图设计"反制点"（如冰原图强控、火山图高火抗），让单一思路在某些图吃瘪。

## 9. 阶段体验节奏设计（Pacing）

| 阶段 | 等级 | 时长预期 | 核心体验 | 情感曲线 | 系统开放 |
|---|---|---|---|---|---|
| 新手期 | L1–10 | 30~60 min | 教学、快速正反馈、首杀 Boss | 好奇→小成就 | 战斗/生存分支、基础装备、首核(地) |
| 成长期 | L11–22 | 2~4 h | 主线反转、加点/配装决策、多阶段 Boss | 紧张→震撼(五处反转) | 精通分支、四核集齐、强化、词缀 |
| 精炼期 | L23–50 | 8~20 h | 刷装/词缀/强化、Build 成型、转生准备 | 掌控→追求完美 | 深渊、Meta 雏形、洗点自由 |
| 终局 | L50+/转生 | 数周 | 赛季榜、无限层、BD 竞争、长线钩子 | 成就感→社交比较→新赛季 | 赛季系统、神话 sink、排行榜 |

**节奏原则**：
- 新手期"死亡几乎无代价"，降低劝退；成长期开始"死亡有代价"，逼迫正视走位与配装。
- 每个关键节点（章节完成/Boss 倒下/核心入手）都有 flavor 短句 + 钩子，制造"再看一眼"的沉浸。
- 长线靠"可肝的 aspirational 目标 + 赛季重置"维持，而非单纯数值堆高。

## 10. 验证与迭代计划（Playtest）

- **成功标准（每次测试前先定义）**：
  - 新手期 30 分钟内可独立通关幕0–1（无设计师协助）。
  - 任意单一 Build 在「全图混编」测试中胜率 < 85%（证明无最优解）。
  - 强化 +15 平均耗时 / 金币消耗落在 `[PLACEHOLDER]` 区间（sink 有效但不过苛）。
  - 普通难度下，同级玩家 vs Boss 的 TTK 在 20~45s（爽但不无脑）。
- **观察 vs 解读分离**：playtest 记录只写"发生了什么"，结论单独标注"这意味着什么"。
- **早期优先修手感（hit-stop/跳字/反馈），后修数值**——先确保循环本身有趣。

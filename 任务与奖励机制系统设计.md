# 任务与奖励机制系统设计文档（GDD v1）

> 适用项目：传奇 H5 ARPG（Canvas 2D 单文件 / 可转微信小游戏）
> 设计目标：用「内在动机」而非「打卡疲劳」驱动玩家持续完成任务。
> 单一数据源：`BAL` 常量（见 `index.html`）；任务引擎：`quest.js` + `quest_defs.json`。
> 本文所有数值均挂 `BAL` 与既有的 ×7 经济（`REWARD_SCALE:7`），未实测项标 `[PLACEHOLDER]`。

---

## 0. 设计原则（先立规矩，再填数值）

我们不希望玩家把任务当成「上班打卡」。三条硬原则：

1. **自决理论（SDT）三支柱**——所有任务设计都要至少命中一项：
   - **胜任感（Competence）**：任务难度始终落在「踮脚够得着」区间；成就感来自「我变强了 / 我秀了一把」。
   - **自主性（Autonomy）**：每日 / 周常有可选池，玩家挑想做的；成就提供长期方向但不强制。
   - **关系感（Relatedness）**：组队任务、排行榜、互助，把单人刷怪变成社交事件。
2. **奖励绑定「怎么玩」而非「玩了没」**——部分任务奖励随战斗深度（连招 / 元素反应 / 弱点命中 / 破防处决）浮动，直接呼应《战斗系统深化设计》里「无聊是因为没有决策」的诊断。
3. **防通胀是设计义务**——任何奖励发放都要有对应 sink；重复任务奖励随玩家等级与内容适配度衰减；活动代币强制周清零（既制造紧迫感，又杜绝囤积通胀）。

## 1. 任务类型总览（四层 + 交叉）

| 类型 | 标识 | 重置周期 | 核心作用 | 难度来源 | 奖励主基调 |
|---|---|---|---|---|---|
| 主线任务 | `main` | 不可逆、链锁 | 叙事推进 + 地图 / 系统解锁 | 固定（按幕递进） | 大额 EXP + 技能点 + 解锁 + 剧情反馈 |
| 每日任务 | `daily` | 每日 0 点 | 日活钩子 + 习惯养成 | 玩家自选 D1–D5 | 活动代币 + 中量 EXP/金币 + 幸运箱 |
| 周常任务 | `weekly` | 每周一 0 点 | 周留存 + 社交（多为组队） | D3–D5 为主 | 周榜积分 + 稀有材料 + 组队加成 |
| 成就任务 | `achievement` | 永久 | 长线目标 + 身份认同（称号） | 累积型 / 挑战型 | 称号 + 永久属性章 + 成就点 |

> **与现有引擎的对接**：`main` 即既有 `arcs[].chapters[]`（已含 `prereqArc/prereqChapter`）。新增 `dailyPool / weeklyPool / achievements` 三个顶层数组，复用同一套 `objective` 结构与状态机（`locked→available→active→completed`）。

## 2. 难度分级与完成条件

### 2.1 五档难度（D1–D5）

| 档位 | 名称 | 预期时长 | 推荐等级匹配 | 死亡惩罚 | 奖励乘子 `DIFF` |
|---|---|---|---|---|---|
| D1 | 闲庭 | 1–3 min | 低于当前 3+ 级 | 无 | ×0.6 |
| D2 | 寻常 | 3–8 min | ±2 级 | 无 | ×1.0 |
| D3 | 险阻 | 8–15 min | 等同当前 | 轻（掉落减半） | ×1.6 |
| D4 | 凶险 | 15–30 min | 高于当前 2–4 级 | 中（掉装备 1 件） | ×2.6 |
| D5 | 噩梦 | 30+ min | 高于当前 5+ 级 / 需组队 | 重（掉装备 + 冷却） | ×4.0 |

> 难度乘子作用于 **EXP / 金币 / 代币 / 材料数量**，不对「解锁」「称号」生效。

### 2.2 完成条件与校验方式

每个 objective 必须可程序化校验（`quest.js` 事件总线已支持）：

- `reach`（到达坐标 / 地图）、`kill`（击杀指定 target 数量）、`collect`（拾取 / 背包持有）、`talk`（与 NPC 对话）、`boss`（击杀 Boss）、`explore`（点亮区域 / 触发点）、`escort`（护送目标存活到达）。
- **新增校验类型**（支撑深度战斗任务）：
  - `combo`：单场达成连击 ≥ N（呼应连招系统）。
  - `react`：触发指定元素反应 ≥ N 次（火+冰 等，呼应《战斗系统深化设计》）。
  - `break`：造成破防 / 处决 ≥ N 次。
  - `weak`：弱点命中率 ≥ X%。
  - `team`：在 ≥2 人队伍中完成（组队任务）。

## 3. 任务链与前置依赖

### 3.1 依赖模型

```
task = {
  id, type, difficulty,
  prereq: [taskId...],      // 前置任务（全完成才 available）
  minLevel,                  // 等级门槛（复用 MON 分层逻辑）
  mapLock,                  // 地图门槛（如 D5 周常锁在魔龙深渊）
  repeat: 'none'|'daily'|'weekly',
  objectives: [...],
  rewards: {...},
  onCompleteFeedback: '...' // 复用既有浮字反馈
}
```

- **主线链**：既有 `prereqArc/prereqChapter` 已支持线性链；新增 `branches`（已在 `arc0_ch2/arc4_ch2/arc5_ch2` 落地）做叙事分叉，分叉汇流回同一终章。
- **每日 / 周常**：**横向可选池 + 纵向解锁树**。例：周常「组队讨伐·初」无前置；完成后解锁「组队讨伐·极」（D5，掉神话材料）。
- **成就链**：累积型（击杀数 / 探索度）→ 挑战型（无伤通关某 Boss / 单场连招 50）→ 荣誉型（全成就达成给专属称号）。

### 3.2 引导节奏（循序渐进）

| 玩家阶段 | 开放任务 | 目的 |
|---|---|---|
| 新手（L1–5） | 主线 + 1–2 个 D1 每日 | 习惯战斗 / 拾取，不被任务压垮 |
| 成长期（L6–15） | 主线 + D1–D3 每日 + 基础成就 | 建立日活，开始有死亡代价 |
| 精炼期（L16–25） | 全每日 + 周常（含组队）+ 进阶成就 | 社交与 Build 成型 |
| 终局（L26+ / 转生） | 周常 D4–D5 + 荣誉成就 + 转生专属循环任务 | 长线目标，反单调 |

## 4. 奖励结构（多元化 + 阶梯递增）

### 4.1 奖励类型清单

| 类型 | 字段 | 实际游戏价值 | 主要 sink（防通胀） |
|---|---|---|---|
| 经验值 | `exp` | 升级 → 属性 / 技能点 | 等级上限 + 转生回收 |
| 金币 | `gold` | 购买基础物资 / 强化 | 强化石消耗 / NPC 回购税 |
| 活动代币 | `token` | 兑换周常商店（限时外观 / 材料） | **每周清零**（强制周转） |
| 强化石 | `stone` | 装备 +1~+15 | 强化失败损耗 |
| 稀有材料 | `material` | 套装 / 词缀重铸 | 重铸消耗 |
| 稀有道具 | `item` | 装备 / 消耗品 | 装备绑定 / 消耗 |
| 技能点 | `skillPoints` | 解锁技能树节点 | 树满即止 |
| 称号 | `title` | 身份 + 微量永久属性章 | 不可交易（荣誉 sink） |
| 成就点 | `achPoints` | 成就榜排名 + 里程碑奖励 | 上限随版本 |
| 转生点 | `rebirth` | 转生后永久增益 | 仅转生获取 |

### 4.2 阶梯奖励公式（关键：与成长曲线匹配）

**经验奖励随玩家等级沿 `BAL` 升级曲线缩放**，保证「每日任务始终约等于恒定比例的一级」，既不前期过载、也不后期无意义：

```
expToNext(L) = BAL.EXP_BASE * BAL.EXP_GROWTH^(L-1)     // 150 * 1.32^(L-1)
questExp = ROUND( BASE_EXP[difficulty] * expToNext(playerLevel) / expToNext(1) * qualityBonus )
         = ROUND( BASE_EXP[d] * 1.32^(playerLevel-1) * qualityBonus )
```

- `BASE_EXP`（D1→D5）：`[30, 60, 120, 240, 480]`（这是 L1 基准值，实际随等级放大）。
- `qualityBonus`：见 §5 随机 / 表现加成，范围 `0.5 ~ 3.0`。

**金币 / 代币同步缩放**（金币取曲线一半增速，避免与 EXP 等比膨胀）：

```
questGold   = ROUND( BASE_GOLD[d] * 1.18^(playerLevel-1) )      // [40,80,160,320,640]
questToken  = BASE_TOKEN[d]                                       // [5,10,20,40,80] 不随等级膨胀（代币本身周清零）
```

> 设计意图：EXP 跟随升级曲线（保证「每天推进约半级」的体感恒定）；金币慢于曲线（后期靠刷怪 / 交易而非任务）；代币恒定（周清零，纯周转货币）。

### 4.3 阶梯递增示例（L10 玩家，D2 任务，qualityBonus=1.0，已用 `BAL` 曲线核验）

- `expToNext(10) = 150 * 1.32^9 ≈ 1825`
- `questExp = 60 * 1.32^9 ≈ 730`（≈ 0.4 级，合理日活剂量）
- `questGold = 80 * 1.18^9 ≈ 355`
- `questToken = 10`

D5 同玩家：`questExp ≈ 730 × (480/60) × 4.0 ≈ 23,360`（≈ 12.8 级，对应噩梦高风险高回报）。

## 5. 随机奖励与惊喜机制（可变比率强化）

行为经济学铁律：**可变比率奖励（variable-ratio schedule）是最高效的动机引擎**。固定奖励会让人「算完就走」，随机奖励让人「再来一次」。

### 5.1 幸运箱（每次任务完成必roll）

完成任意任务后，按权重 roll 一个幸运箱品质（复用 `QUALITY` 掉落权重体系）：

| 品质 | 权重（基础） | 内容物 |
|---|---|---|
| 普通 | 600 | 少量金币 |
| 优秀 | 250 | 金币 + 1 强化石 |
| 稀有 | 100 | 材料 + 概率技能点 |
| 史诗 | 38 | 稀有材料 + 高概率代币 |
| 传说 | 10 | 稀有装备（按玩家等级） |
| 神话 | 2 | 传说装备 / 专属外观碎片 |

### 5.2 保底 /  pity（防失望，守住正反馈）

- **软保底**：连续 20 次未出 ≥史诗，第 21 次起史诗权重 ×3；连续 50 次未出传说，下次传说权重 ×5。
- **硬保底**：每完成 100 个任务，必出 1 个传说箱（成就型里程碑反馈）。
- 设计理由：纯随机会偶发长旱引发弃游；保底把「最坏体验」封顶，同时不破坏惊喜感。

### 5.3 惊喜掉落（Jackpot）

- 每日首次登录触发 **幸运转盘**（免费 1 次），奖励在 `[金币小、代币中、材料、外观碎片、空]` 间随机——「每日都有小期待」。
- 周常 D4/D5 完成有 `3% [PLACEHOLDER]` 概率额外掉 **惊喜宝箱**（奖励 ×5 量级的「横财」），制造可传播的「欧皇时刻」。
- 成就达成瞬间播放专属动画 + 全服通报（荣誉 + 社交证明，呼应 Cialdini 的「社会认同」）。

### 5.4 表现加成（把随机和「技术」挂钩）

部分任务 `qualityBonus` 由战斗深度决定，让随机奖励奖励「秀操作」而非纯运气：

```
qualityBonus = 1.0
  + 0.3 * clamp(comboMax/30, 0, 1)        // 高连招
  + 0.3 * clamp(reactionCount/5, 0, 1)     // 元素反应
  + 0.4 * clamp(breakExecute/3, 0, 1)      // 破防处决
  + 0.5 * weakHitRate                        // 弱点命中率
  + (teamQuest ? 0.2 : 0)                  // 组队默契
上限 3.0
```

> 这直接解决「无聊砍杀」：同一任务，无脑刷拿基础箱，秀操作拿神话箱——**过程本身就分高下**。

## 6. 进度可视化与实时反馈

复用并扩展 `quest.js` 的 HUD 追踪栏与 `onCompleteFeedback` 浮字。

### 6.1 四层可视

1. **HUD 追踪栏（常驻）**：当前主线 + 至多 3 条活跃每日 / 周常目标，每行 `[✓/◯] 描述 进度(12/20)`，进度条实时填充。
2. **任务日志面板（Tab 键 / 按钮）**：四个标签页（主线 / 每日 / 每周 / 成就），成就页按类别 + 进度条展示。
3. **目标达成反馈**：objective 完成的瞬间，该行打勾 + 轻微高亮 + 音效（复用既有 `click/levelup` 音效体系）；章节完成弹 `onCompleteFeedback` 浮字（💬）。
4. **奖励发放动画**：领取时道具图标飞入背包 + 跳字（`+852 EXP`、`+10 代币`、`★传说箱！`），神话 / 惊喜掉落触发屏幕级特效 + 顿帧（复用 `hit-stop`）。

### 6.2 实时反馈循环

```
玩家动作 → 事件总线 emit('objective:progress', {id, delta})
        → quest.js 更新 state + 刷新 HUD 行
        → 若达 amount → emit('objective:done') → 打勾动画
        → 全部 objective 完成 → emit('chapter:done') → 浮字 + 幸运箱 roll + 奖励动画
```

> 关键：反馈延迟必须 < 100ms，否则「我做对了」的认知链断裂，动机流失。

## 7. 社交奖励元素

### 7.1 排行榜（竞争 + 社会认同）

| 榜单 | 排序依据 | 更新 | 奖励 |
|---|---|---|---|
| 等级榜 | 角色等级 | 实时 | 周前 10% 得专属称号 |
| 战力榜 | 综合战力（属性+装备+核心） | 实时 | 周前 3% 得限时光效 |
| 成就榜 | 成就点 | 每日 | 里程碑外观 |
| 速通榜 | 主线通关时长 | 事件 | 荣誉称号「破晓者」 |
| 组队贡献榜 | 周常组队次数 / 协助击杀 | 每周 | 团队代币分红 |

> 设计：榜单只给**外观 / 称号 / 限时**奖励（不稀释核心数值经济），既制造竞争又不破坏平衡；每周重置避免「早期玩家永久霸榜」劝退新人（Cialdini 稀缺 + 重承诺）。

### 7.2 组队任务与默契

- 周常 `team` 类任务强制 ≥2 人（部分 D5 需 ≥3）。
- **组队加成**：队伍每人奖励 ×1.15；若含「首次同队」玩家额外 ×1.05（鼓励拓展社交圈）。
- **默契值（Sunk Cost / Commitment）**：与固定队友累计完成周常，默契等级提升，解锁共享仓库格 / 组队专属表情 —— 用沉没成本 + 承诺装置留住核心社交关系。

### 7.3 互助与公会

- **互助任务**：向好友发送 / 领取「体力 / 材料」小礼，双方得微量代币（病毒式拉新 + 关系感）。
- **公会建设**：公会任务产出「公会贡献」，贡献换公会 Buff（全团经验 +5% [PLACEHOLDER]），制造集体目标。

## 8. 与经济系统的对接（防通胀，硬约束）

复用《游戏机制与系统设计》的 sources/sinks 框架。任务系统是 **sources**，必须有对应 **sinks**：

| 奖励 | 主要 sink | 通胀监测指标 |
|---|---|---|
| 金币 | 强化石购买 / 重铸税 / NPC 回购折损 | 人均日金币净增 > 设置阈值 → 触发平衡 pass |
| 代币 | **每周清零** + 周常商店限时兑换 | 代币存量随周归零，天然无通胀 |
| 强化石 | 强化失败损耗（已设计 +15 曲线） | 石价 / 产出比监控 |
| 材料 | 套装合成 / 词缀重铸 | 材料囤积上限 |
| 装备 | 绑定 / 分解（返少量石） | 神话掉率恒 2/1000 权重，不随产出放大 |

**衰减规则（防「低端内容刷到毕业」）**：

```
if playerLevel - questRecommendedLevel >= 5:
    questExp   *= 0.4     // 碾压旧内容，经验衰减
    questGold  *= 0.6
    token       不变（仍给，但周清零）
```

> 这条直接杜绝「回新手村刷每日毕业」的通胀漏洞。

### 10.1 `quest_defs.json` 扩展 schema

```jsonc
{
  "version": 2,
  "schema": { /* 既有 + 新增 */ "questType":["main","daily","weekly","achievement"],
               "difficulty":["D1","D2","D3","D4","D5"] },
  "arcs": [ /* 既有主线，不变 */ ],
  "dailyPool": [
    { "id":"daily_d2_kill","difficulty":"D2","weight":100,
      "objectives":[{"type":"kill","target":"slime","amount":20,"desc":"击退 20 只史莱姆"}],
      "rewards":{"exp":60,"gold":80,"token":10},
      "luckyBox":true }
  ],
  "weeklyPool": [
    { "id":"weekly_team_boss","difficulty":"D4","team":true,"minLevel":16,
      "objectives":[{"type":"boss","target":"冰霜巨魔","amount":1},{"type":"team","amount":2}],
      "rewards":{"exp":240,"token":40,"material":2},"prereq":["weekly_team_intro"] }

  "achievements": [
```

### 10.2 分阶段实现

- **P0（动机兜底）**：每日 D1–D2 池 + 幸运箱（基础权重）+ HUD 追踪栏扩展 + 完成浮字。立刻让「上线有事儿干」。
- **P1（深度挂钩）**：`combo/react/break/weak` 校验类型 + `qualityBonus` 表现加成 + 成就系统 + 称号。把奖励和「怎么打」绑死。
- **P2（社交）**：周常 `team` 任务 + 组队加成 + 排行榜（前端读 `account` 服务端数据）+ 互助。
- **P3（经济闭环）**：代币周清零 + 碾压衰减 + pity 保底 + 惊喜箱 + 通胀监控告警。

## 12. 一句话总结

任务系统不是「待办清单」，而是**把玩家的每一个动机开关（胜任 / 自主 / 关系）接到对应的奖励与反馈上**：主线给意义，每日给习惯，周常给社交，成就给身份；随机奖励给期待，表现加成给尊重，社交榜给荣耀——且每一份奖励都有 sink，经济不崩，玩家不腻。
